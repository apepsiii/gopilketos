# Deployment Guide - Pilketos E-Voting

## Prerequisites

- Go 1.21+ installed locally (for building)
- SSH access to VPS
- Domain pointing to VPS IP

---

## Quick Deployment

### Step 1: Build

```bash
chmod +x build.sh
./build.sh
```

### Step 2: Upload to VPS

```bash
scp -r dist/* user@your-vps-ip:/opt/pilketos/
```

### Step 3: Run Installer

```bash
ssh user@your-vps-ip
cd /opt/pilketos
chmod +x pilketos-setup
sudo ./pilketos-setup
```

The wizard will:
1. Ask for your domain (e.g., vote.bersekola.app)
2. Ask for port (default: 8024)
3. Ask for install directory (default: /opt/pilketos)
4. Ask for admin username (default: admin)
5. Ask for admin password (auto-generated if empty)
6. Show progress bar during installation
7. Display success message with login info

---

## What the Installer Does

1. Creates installation directories
2. Copies the pilketos binary
3. Generates config.yaml
4. Sets file permissions
5. Creates systemd service
6. Configures Nginx reverse proxy
7. Starts and enables the service

---

## After Installation

Access your application:
- URL: `https://your-domain.com`
- Admin: `https://your-domain.com/admin/login`

Manage the service:
```bash
sudo systemctl status pilketos
sudo systemctl restart pilketos
sudo journalctl -u pilketos -f
```

## Manual Setup (Without Wizard)

If the wizard fails, you can do it manually:

### 1. Install Dependencies

```bash
sudo apt update
sudo apt install nginx certbot python3-certbot-nginx -y
```

### 2. Create Directory

```bash
sudo mkdir -p /opt/pilketos/database
```

### 3. Copy & Configure

Copy `pilketos` binary to `/opt/pilketos/`

Create `/opt/pilketos/config.yaml`:
```yaml
app_name: pilketos
port: "8024"
domain: vote.bersekola.app
install_dir: /opt/pilketos
admin_user: admin
admin_pass: your_secure_password
db_path: /opt/pilketos/database/evoting.db
```

### 4. Set Permissions

```bash
sudo chmod +x /opt/pilketos/pilketos
```

### 5. Systemd Service

Create `/etc/systemd/system/pilketos.service`:
```ini
[Unit]
Description=Pilketos E-Voting System
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/pilketos
ExecStart=/opt/pilketos/pilketos
Restart=always
RestartSec=5
StandardOutput=append:/var/log/pilketos.log
StandardError=append:/var/log/pilketos_error.log

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable pilketos
sudo systemctl start pilketos
```

### 6. Nginx Config

Create `/etc/nginx/sites-available/vote.bersekola.app`:
```nginx
server {
    listen 80;
    server_name vote.bersekola.app;

    location / {
        proxy_pass http://127.0.0.1:8024;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/vote.bersekola.app /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx
```

### 7. SSL

```bash
sudo certbot --nginx -d vote.bersekola.app
```

## Troubleshooting

### Service won't start
```bash
sudo journalctl -u pilketos -n 50
```

### Port already in use
```bash
sudo lsof -i :8024
```

### Nginx 502 Bad Gateway
- Check pilketos is running: `sudo systemctl status pilketos`
- Test directly: `curl http://localhost:8024`

### SSL certificate issues
```bash
sudo certbot renew --dry-run
```

## File Structure

```
/opt/pilketos/
├── pilketos          # Main binary (standalone with embedded assets)
├── pilketos-setup   # Installer wizard
├── config.yaml      # Configuration (generated by installer)
└── database/        # SQLite database (created automatically)
```
